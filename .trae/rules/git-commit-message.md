# 提交说明规范（Conventional Commits）
# 格式：<type>(<scope>): <subject>
# 空一行
# <body>
# 空一行
# <footer>

# -------------------------- 类型（type）可选值 --------------------------
# feat:     新功能
# fix:      修复 bug
# docs:     文档变更
# style:    代码格式（不影响逻辑，如空格、分号）
# refactor: 重构（不新增功能、不修复 bug）
# perf:     性能优化
# test:     新增/修改测试
# build:    构建系统/依赖变更
# ci:       CI 配置、脚本变更
# chore:    杂项（工具、配置等，不影响业务代码）
# revert:   回滚某次提交

# -------------------------- 范围（scope）示例 --------------------------
# 模块：order、asset、user、auth、ui、api、db
# 文件：transaction、factory、crud、service

# -------------------------- 标题（subject）要求 --------------------------
# - 不超过 50 字符
# - 祈使句、现在时态
# - 首字母小写，结尾无句号

# -------------------------- 正文（body）说明 --------------------------
# - 说明「为什么改、改了什么、影响什么」
# - 每行不超过 72 字符
# - 可多行

# -------------------------- 脚注（footer）说明 --------------------------
# - 关联 Issue：Fixes #123
# - 不兼容变更：BREAKING CHANGE: 说明
# - 其他：Ref: #123、Co-authored-by: xxx
