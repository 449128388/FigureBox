## 修复删除订单500错误

### 问题描述
删除订单时接口返回 500 Internal Server Error。

### 问题原因
`AssetTransaction` 和 `OrderTransaction` 表都有 `order_id` 外键关联到 `orders` 表。当删除订单时：
1. 外键约束阻止了订单的物理删除
2. 软删除交易记录时 `order_id` 外键仍然存在

错误信息：
```
Cannot delete or update a parent row: a foreign key constraint fails
(`figurebox`.`asset_transactions`, CONSTRAINT `asset_transactions_ibfk_3` 
FOREIGN KEY (`order_id`) REFERENCES `orders` (`id`))
```

### 解决方案
**`backend/app/api/orders.py`** `delete_order` 方法：

1. **解除外键关联**：将交易记录的 `order_id` 设为 `None`
2. **添加 `synchronize_session=False`**：确保 SQLAlchemy 正确执行更新
3. **分步提交**：先提交交易记录更新，再删除订单

```python
# 【修复】软删除关联的资产交易记录（库存账）
db.query(AssetTransaction).filter(
    AssetTransaction.order_id == order_id
).update({
    'is_active': False,
    'deleted_at': datetime.now(),
    'order_id': None  # 【修复】解除外键关联，避免外键约束错误
}, synchronize_session=False)

# 【修复】软删除关联的资金流水记录（资金账）
db.query(OrderTransaction).filter(
    OrderTransaction.order_id == order_id
).update({
    'is_active': False,
    'deleted_at': datetime.now(),
    'order_id': None  # 【修复】解除外键关联，避免外键约束错误
}, synchronize_session=False)

db.commit()  # 【修复】先提交交易记录的更新

db.delete(db_order)
db.commit()
```

### 修复效果
- ✅ 订单删除功能正常工作
- ✅ 关联的交易记录被正确软删除（保留历史数据）
- ✅ 外键约束不再阻止删除操作
- ✅ 保持数据一致性

## 修复手办编辑时缺少资金流水记录

### 问题描述
在手办编辑时，当数量和入手价格发生变动时，`order_transactions` 表（资金账）中缺少与 `asset_transactions` 表（库存账）对应的资金流水记录。

### 问题原因
编辑手办接口只创建了 `asset_transactions` 记录（库存变动），但没有创建对应的 `order_transactions` 记录（资金变动），导致库存账和资金账不一致。

### 解决方案

**1. `backend/app/services/order_transaction_service.py`**：
新增两个方法用于创建冲正交易的资金流水记录：
- `create_quantity_adjustment_transaction()` - 数量调整时的资金流水
- `create_price_adjustment_transaction()` - 价格调整时的资金流水

**2. `backend/app/api/figures.py`**：
在编辑手办时，同时创建库存账和资金账记录：
```python
# 数量变化时
if adjustment_price > 0:
    # 1. 创建资产交易记录（库存账）
    AssetTransactionService.create_quantity_adjustment_transaction(...)
    # 2. 创建资金流水记录（资金账）
    OrderTransactionService.create_quantity_adjustment_transaction(...)

# 价格变化时
if new_price != original_price:
    # 1. 创建资产交易记录（库存账）
    AssetTransactionService.create_price_adjustment_transaction(...)
    # 2. 创建资金流水记录（资金账）
    OrderTransactionService.create_price_adjustment_transaction(...)
```

### 修复效果
- ✅ 数量增加时：同时记录库存增加和资金支出
- ✅ 数量减少时：同时记录库存减少和资金冲正（负金额）
- ✅ 价格调整时：同时记录库存价格调整和资金差额
- ✅ 库存账和资金账保持一致

## 修复价格调整不应记录到库存账

### 问题描述
在编辑手办时，如果数量不发生变动但入手价格发生变动，会创建一条 `transaction_type="adjust"` 的记录到 `asset_transactions`（库存账）中。

这会导致库存统计错误：
- 如果包含 adjust 记录，总数量 = 5 + 2 + 7 = 14（实际只有7个）
- 如果排除 adjust 记录，总数量 = 7 但总价值不对

### 问题原因
库存账（`asset_transactions`）应该只记录数量变动，价格调整不影响库存数量，不应该记录到库存账中。

### 解决方案
**`backend/app/api/figures.py`**：
修改价格调整时的处理逻辑，只创建资金流水记录（`order_transactions`），不再创建库存账记录（`asset_transactions`）：

```python
# 检查价格变化（数量未变化时）
if ('purchase_price' in figure_data and
    new_price != original_price and
    new_quantity == original_quantity):
    # 【修复】价格调整只创建资金流水记录（资金账），不创建库存账记录
    # 因为库存账只记录数量变动，价格调整不影响库存数量
    OrderTransactionService.create_price_adjustment_transaction(
        db=db,
        user_id=current_user.id,
        figure_id=figure_id,
        old_price=original_price,
        new_price=new_price,
        quantity=original_quantity
    )
    db.commit()
```

### 修复效果
- ✅ 价格调整时只记录资金变动，不影响库存数量统计
- ✅ 库存账只记录数量变动（买入、补录、冲正）
- ✅ 资金账记录所有资金变动（买入、补录、冲正、价格调整）
- ✅ 库存统计更准确

## 重构 order_transactions（资金账）核心逻辑

### 重构背景
原 `order_transactions` 存在概念混淆问题：
- 记录了无真实资金流动的场景（库存调整、价格调整）
- 字段设计不够清晰，缺少资金流向标识
- 与库存账职责边界模糊

### 核心原则
**只记录有真实资金流动的交易**：

| 场景 | 资金账记录 | 示例 |
|------|-----------|------|
| **订单支付**（买入） | `trans_type='buy'`, `direction='out'` | 支付宝支付¥800买手办 |
| **订单收款**（卖出） | `trans_type='sell'`, `direction='in'` | 闲鱼收款¥1,200卖出 |
| **定金支付** | `trans_type='buy'`, `direction='out'` | 支付定金¥200预定 |
| **尾款支付** | `trans_type='buy'`, `direction='out'` | 补尾款¥600 |
| **退款**（退货/取消） | `trans_type='refund'`, `direction='in'` | 商家退款¥800 |
| **平台手续费** | `trans_type='fee'`, `direction='out'` | 闲鱼扣手续费¥12 |

**严禁记录的场景（无资金流动）**：

| 场景 | 错误做法 | 正确做法 |
|------|---------|---------|
| **库存调整**（纠错） | ❌ 记 amount=1600 | ✅ **不记**，库存账处理 |
| **冲正库存**（数量减少） | ❌ 记 amount=-2400 | ✅ **不记**，库存账用 adjust |
| **价格调整**（估值变化） | ❌ 记 amount=70 | ✅ **不记**，资产重估非现金流 |
| **自动创建手办**（无订单） | ❌ 预占 amount=800 | ✅ **不记**，等补录订单后再记 |

### 模型字段重构

**`backend/app/models/asset.py`**：

```python
class OrderTransaction(Base):
    # 交易信息
    transaction_type = Column(String(50), nullable=False)  # buy/sell/refund/fee
    direction = Column(String(10), nullable=False, default="out")  # in/out
    quantity = Column(Integer, default=1)
    unit_price = Column(Float, nullable=False)  # 原 price 改为 unit_price
    total_amount = Column(Float, nullable=False)
    currency = Column(String(10), default="CNY")
    
    # 新增字段
    payment_method = Column(String(50))  # 支付宝/微信/银行卡
    platform = Column(String(50))  # 淘宝/闲鱼/AmiAmi/京东/线下
    
    # 时间字段重构
    transaction_date = Column(DateTime(timezone=True), nullable=False)  # 业务时间
    created_at = Column(DateTime(timezone=True), server_default=func.now())  # 系统时间
```

### 服务层重构

- `create_buy_transaction()` - 创建买入记录（direction='out'）
- `create_sell_transaction()` - 创建卖出记录（direction='in'）
- `create_refund_transaction()` - 创建退款记录（direction='in'）
- `create_fee_transaction()` - 创建手续费记录（direction='out'）

删除方法：
- `create_transaction_from_figure()` - 自动创建手办不再记录
- `create_quantity_adjustment_transaction()` - 数量调整不记录
- `create_price_adjustment_transaction()` - 价格调整不记录

### API 调用更新

**`backend/app/api/figures.py`**：

- 移除价格调整时的资金账记录
