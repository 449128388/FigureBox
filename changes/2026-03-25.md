# 2026-03-25 更新日志

## 1. 修复订单出荷日期少一天问题

### 问题原因
- 前端使用 `date.toISOString().split('T')[0]` 将日期转换为字符串
- `toISOString()` 方法返回的是 UTC 时间（协调世界时）
- 中国处于 UTC+8 时区，当选择日期后转换为 UTC 时，时间会回退8小时
- 例如：选择 2026-03-25，转换为 UTC 后变成 2026-03-24 16:00:00，截取日期部分就变成 2026-03-24

### 变更内容

#### 前端更新
**文件**: `frontend/src/views/Orders.vue`
- 修改 `formatDate` 函数的实现
- 原实现：`return date.toISOString().split('T')[0]`
- 新实现：使用 `getFullYear()`, `getMonth()`, `getDate()` 获取本地时间年月日，避免时区转换问题

### 修复代码
```javascript
const formatDate = (date) => {
  if (!date) return null
  if (typeof date === 'string') return date
  // 转换为YYYY-MM-DD格式，使用本地时间避免时区问题
  const d = new Date(date)
  const year = d.getFullYear()
  const month = String(d.getMonth() + 1).padStart(2, '0')
  const day = String(d.getDate()).padStart(2, '0')
  return `${year}-${month}-${day}`
}
```

### 功能效果
- 新增订单时，选择的出荷日期与保存后的日期一致
- 编辑订单时，修改的出荷日期与保存后的日期一致
- 不再出现时区导致的日期偏差问题

### 用户体验优化
- **数据准确**: 用户选择的日期就是保存的日期
- **避免困惑**: 不再出现"明明选了25号，保存后变成24号"的情况
- **时区兼容**: 使用本地时间处理，适应不同时区用户
