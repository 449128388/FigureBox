## 4. 修复手办列表页价格为0时显示"未设置"的问题

### 变更原因
- 手办列表页面中定价显示逻辑为 `figure.price || '未设置'`，当价格为0时，JavaScript将0视为falsy值，导致显示"未设置"
- 入手价格使用 `v-if="figure.purchase_price"` 判断，当价格为0时同样无法显示
- 需要修复显示逻辑，正确区分为0的价格和null/undefined

### 变更内容

#### 前端更新
**文件**: `frontend/src/views/Figures.vue`
- 修改定价显示逻辑：从 `figure.price || '未设置'` 改为 `figure.price !== null && figure.price !== undefined ? figure.price : '未设置'`
- 修改入手价格条件渲染：从 `v-if="figure.purchase_price"` 改为 `v-if="figure.purchase_price !== null && figure.purchase_price !== undefined"`
- 确保价格为0时能正确显示0，而不是"未设置"

### 功能效果
- 当价格为0时，列表页面正确显示0，不再显示"未设置"
- 当价格为null或undefined时，才显示"未设置"
- 价格显示逻辑更加准确，符合用户预期

### 用户体验优化
- **数据准确**: 价格为0时正确显示0，避免用户误解
- **逻辑清晰**: 明确区分为0的价格和未设置的价格
- **信息完整**: 用户可以看到实际的价格数据

## 11. 实现 Token 自动续期机制

### 变更原因
- 原 token 过期时间为 30 分钟，即使用户在操作也会过期
- 用户在使用过程中频繁被踢出需要重新登录，体验差
- 需要实现滑动窗口机制，用户活跃时自动续期 token

### 变更内容

#### 后端更新
**文件**: `backend/app/utils/jwt.py`
- 修改 `verify_token` 函数，返回 `(user_id, should_refresh)` 元组
- 添加 `TOKEN_REFRESH_THRESHOLD_MINUTES = 5` 常量
- 当 token 剩余有效期小于 5 分钟时，标记需要续期

**文件**: `backend/main.py`
- 添加 `TokenRefreshMiddleware` 中间件
- 中间件检查每个请求的 token，如需续期则生成新 token
- 将新 token 添加到响应头 `X-Refresh-Token` 中
- 配置 CORS 暴露 `X-Refresh-Token` 响应头

#### 前端更新
**文件**: `frontend/src/axios/index.js`
- 在响应拦截器中检查 `x-refresh-token` 响应头
- 如果存在新 token，自动更新到 localStorage
- 实现无感知的 token 自动续期

### 功能效果
- 当 token 剩余有效期小于 5 分钟时，系统自动续期
- 用户活跃操作时会自动获得新 token，不会被迫重新登录
- 如果用户 30 分钟无任何操作，token 才会真正过期
- 实现滑动窗口的会话保持机制

### 用户体验优化
- **无感知续期**: 用户操作过程中自动续期，无需手动重新登录
- **会话保持**: 活跃用户不会被意外踢出系统
- **安全可靠**: 长时间无操作后 token 仍会过期，保证安全性
