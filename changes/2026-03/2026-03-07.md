# 2026-03-07 更新日志

## 1. 修复 POST /api/figures/ 接口 422 错误

### 问题
- 调用 `POST /api/figures/` 接口时返回 422 Unprocessable Entity 错误
- 错误详情：`{"detail":[{"type":"date_from_datetime_inexact","loc":["body","release_date"],"msg":"Datetimes provided to dates should have zero time - e.g. be exact dates"...}]}`
- 原因：前端发送的日期格式为 ISO 8601 格式（如 `2026-03-06T16:00:00.000Z`），但后端 Pydantic 模型期望的是纯日期格式（`YYYY-MM-DD`）

### 解决方案
- 在 `FigureBase` 模型中添加 `field_validator` 验证器
- 验证器处理 `release_date` 和 `purchase_date` 字段
- 支持自动解析 ISO 8601 格式的日期时间字符串，提取日期部分
- 同时保持对纯日期字符串的兼容性

### 修改文件
- `backend/app/schemas/figure.py`
  - 导入 `field_validator` 和 `datetime`
  - 添加 `parse_date` 验证器方法，自动转换日期格式

### 修复效果
- POST 接口现在可以正常接收前端发送的 ISO 8601 格式日期
- 日期字段自动转换为纯日期格式存储到数据库
- 接口返回 200 状态码，数据创建成功

## 2. 修复 price 字段验证问题

### 问题
- `FigureCreate` 模型中的 `price` 字段为必填项（`float` 类型）
- 但前端发送的数据中 `price` 可能为 `0` 或空值，导致验证失败

### 解决方案
- 将 `FigureBase` 模型中的 `price` 字段改为可选类型：`price: float | None = None`
- 允许创建手办时不填写价格

### 修改文件
- `backend/app/schemas/figure.py`
  - 修改 `price` 字段类型从 `float` 改为 `float | None = None`

### 修复效果
- 创建手办时价格字段变为可选
- 提升了用户体验，允许先创建手办后补充价格信息

## 3. 删除手办管理模块中的 image_url 字段

### 变更原因
- `image_url` 字段用于存储单张图片URL，但系统已使用 `images` 数组字段存储多张图片
- 两个字段功能重复，`images` 数组更灵活，可以存储多张图片
- 为简化数据模型，决定移除 `image_url` 字段

### 删除内容

#### 后端模型
- `backend/app/models/figure.py`
  - 删除字段：`image_url = Column(String(255))`

#### 后端 Schema
- `backend/app/schemas/figure.py`
  - 从 `FigureBase` 模型删除：`image_url: str | None = None`
  - 从 `FigureUpdate` 模型删除：`image_url: str | None = None`

#### 前端代码
- 经检查，前端代码中未使用 `image_url` 字段，无需修改

### 数据影响
- 数据库中现有数据的 `image_url` 字段将不再被使用
- 建议将现有 `image_url` 数据迁移到 `images` 数组中（如需要）
- 新创建的手办记录将不再包含 `image_url` 字段

### 优化效果
- 数据模型更加简洁，避免字段冗余
- 统一使用 `images` 数组管理手办图片
- 减少数据维护复杂度

## 4. 简化手办管理列表页面展示字段

### 变更原因
- 列表页面展示字段过多，信息密度过大
- 用户只需要关注核心信息：图片、名称、定价、入手价格、入手时间、标签
- 简化展示可以提升页面加载速度和用户体验

### 修改内容

#### 保留字段
- **图片**：展示 `images` 数组的第一张图片（如无图片则显示默认图）
- **名称**：手办名称
- **定价**：手办官方定价
- **入手价格**：实际购买价格
- **入手时间**：购买日期
- **标签**：手办标签列表

#### 删除字段
以下字段不再在列表页面展示：
- `manufacturer`（制造商）
- `release_date`（出货日）
- `purchase_method`（入手途径）
- `purchase_type`（入手形式）
- `scale`（比例）
- `prototype`（原型）
- `painting`（涂装）
- `original_art`（原画）
- `work`（作品）
- `material`（材质）
- `size`（尺寸）

#### 样式优化
- 将编辑和删除按钮放入 `.figure-actions` 容器
- 添加 Flex 布局，按钮平均分布
- 添加顶部边框分隔线，提升视觉层次

### 修改文件
- `frontend/src/views/Figures.vue`
  - 简化 `.figure-item` 中的展示字段
  - 将按钮包裹在 `.figure-actions` div 中
  - 添加 `.figure-actions` CSS 样式

### 优化效果
- 列表页面更加简洁清晰
- 核心信息一目了然
- 页面加载速度提升
- 用户体验改善
