## 3. 添加响应式页面宽度适配

### 变更原因
- 不同分辨率下需要不同的页面宽度以获得最佳显示效果
- 1920×1080 分辨率适合较宽的页面布局（max-width: 1800px）
- 2560×1600 分辨率（150%缩放后视口约1707px）适合较窄的页面布局（max-width: 1600px），避免内容过于分散

### 修改内容

#### frontend/src/App.vue
- 添加 CSS 媒体查询实现响应式宽度适配
- 默认宽度设置（适用于1920×1080分辨率，视口约1920px）：`max-width: 1800px`
- 2560×1600分辨率（150%缩放后视口约1707px）：`max-width: 1600px`
- 使用范围媒体查询 `@media (min-width: 1700px) and (max-width: 1800px)` 精确匹配2560×1600分辨率

```css
/* 默认宽度设置（适用于1920*1080分辨率，视口约1920px） */
#app {
  max-width: 1800px;
  margin-left: 50px;
  margin-right: 50px;
  padding: 20px;
  width: 100%;
}

/* 2560*1600分辨率（考虑150%缩放后视口约1707px）使用1600px宽度 */
/* 1700px-1800px范围是2560×1600分辨率150%缩放后的视口宽度 */
@media (min-width: 1700px) and (max-width: 1800px) {
  #app {
    max-width: 1600px;
  }
}
```

### 适配规则

| 视口宽度 | 页面最大宽度 | 适用场景 |
|----------|-------------|----------|
| < 1700px | 1800px (默认) | 较小屏幕 |
| 1700px - 1800px | 1600px | 2560×1600 (150%缩放) |
| > 1800px | 1800px (默认) | 1920×1080 (100%缩放) |

### 技术要点
- 考虑Windows系统缩放比例对CSS像素的影响
- 2560×1600分辨率在150%缩放后，实际视口宽度约为1707px
- 使用范围媒体查询避免影响1920×1080分辨率（视口约1920px）

### 优化效果
- 不同分辨率下页面显示更加协调
- 1920×1080 分辨率下充分利用屏幕空间（1800px）
- 2560×1600 分辨率下避免内容过于分散，保持紧凑布局（1600px）
- 提升多设备兼容性

## 6. 添加跨分页名称字段定位功能

### 变更原因
- 用户在"作者"、"属性"、"规格"分页填写信息时，可能忘记填写"基础"分页的"名称"字段
- 提交表单时发现名称未填写，需要手动切换回基础分页，体验不佳
- 需要自动定位并聚焦到名称字段，提升用户体验

### 变更内容

#### 1. 标签页配置变更
**文件**: `frontend/src/views/Figures.vue` (第58行及后续)
- 为 `el-tabs` 添加 `v-model="activeTab"` 和 `ref="formTabs"` 属性
- 为各 `el-tab-pane` 添加 `name` 属性：
  - 基础页: `name="basic"`
  - 作者页: `name="author"`
  - 属性页: `name="tags"`
  - 规格页: `name="specs"`

#### 2. 数据层变更
**文件**: `frontend/src/views/Figures.vue` (第274行)
- 在 `data()` 中添加 `activeTab: 'basic'` 用于控制当前激活的标签页

#### 3. 模板层变更
**文件**: `frontend/src/views/Figures.vue` (第73行)
- 为"名称"输入框添加 `ref="nameInput"` 以便聚焦

#### 4. 方法层变更
**文件**: `frontend/src/views/Figures.vue` (新增第447-458行)
- 新增 `focusToNameField()` 方法：
  - 将 `activeTab` 设置为 `'basic'`，自动切换到基础分页
  - 使用 `this.$nextTick()` 等待DOM更新
  - 调用 `this.$refs.nameInput.focus()` 聚焦到名称输入框

- 修改 `validateNameOnInput()` 方法：
  - 当名称为空时，调用 `focusToNameField()` 自动跳转并聚焦
  - 当验证失败（长度或字符不符合要求）时，同样调用 `focusToNameField()`

### 功能效果
- **自动跳转**: 当用户在非基础分页时触发名称验证失败，自动切换回基础分页
- **自动聚焦**: 页面切换完成后，自动将光标聚焦到名称输入框
- **即时反馈**: 用户无需手动切换分页和点击输入框，直接可以输入
- **适用场景**: 
  - 用户在"作者"、"属性"、"规格"分页点击提交按钮
  - 用户在这些分页尝试输入内容时触发表单验证

### 技术实现
- 使用 Vue 的 `$nextTick()` 确保DOM更新完成后再执行聚焦操作
- 使用 Element Plus 的 `v-model` 绑定实现标签页切换
- 使用组件的 `ref` 引用实现输入框聚焦
