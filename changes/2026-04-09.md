- **修复指数数据重复插入问题**：
  - 问题：在 `stock_index_history` 表中观察到同一指数在同一秒内插入多条重复数据
    - 上证指数示例：两条 sh000001 数据，request_time 分别为 17:06:27 和 17:06:29（仅相差2秒）
    - 沪深300示例：两条 sh000300 数据，request_time 分别为 17:06:34 和 17:06:35（仅相差1秒）
  - 根因分析：
    1. `get_dashboard_data()` 函数同时调用 `get_cached_sh_index()` 和 `get_cached_hs300_index()`
    2. 两个函数独立执行缓存检查，都判断为 `need_fetch = True`
    3. 两个函数都满足 `request_count < 6` 条件
    4. 结果：同一API请求中，两个函数都向腾讯财经发起请求并插入历史记录
  - 影响：
    - `stock_index_history` 表：存在重复数据，浪费存储空间
    - `stock_index_cache.request_count`：会错误地增加多次计数
    - 每日限流（6次）：会提前耗尽，影响后续请求
  - 解决方案：
    1. 添加全局锁机制：使用 `_index_fetch_locks` 字典 + `_index_fetch_locks_lock` 线程锁
    2. 锁键设计：`{index_code}_{YYYYMMDDHHMMSS}`（精确到秒）
    3. 防重复逻辑：
       - 同一秒内已有请求在获取数据时，后续请求等待0.5秒后使用缓存数据
       - 使用 `try...finally` 确保锁的清理
    4. 同时修复上证指数和沪深300的缓存函数
  - 修改文件：
    - `backend/app/api/assets.py`
