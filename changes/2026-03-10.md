# 2026-03-10 更新日志

## 1. 修复"名称"字段校验问题

### 问题描述
- 在"名称"字段中输入包含中文字符的内容（如"我回复挂号费的"）时，前端错误提示"名称不能只包含特殊字符"
- 原因：emoji过滤的正则表达式范围设置错误，包含了中文字符的范围，导致中文字符被错误过滤

### 解决方案

#### 1. 代码修改
**文件**: `frontend/src/views/Figures.vue` (第396行)
- 修正 emoji 过滤的正则表达式：
  ```javascript
  // 原代码（错误）
  const emojiPattern = /[\u{1F600}-\u{1F64F}\u{1F300}-\u{1F5FF}\u{1F680}-\u{1F6FF}\u{1F1E0}-\u{1F1FF}\u{2702}-\u{27B0}\u{24C2}-\u{1F251}]/gu
  
  // 修正后（正确）
  const emojiPattern = /[\u{1F600}-\u{1F64F}\u{1F300}-\u{1F5FF}\u{1F680}-\u{1F6FF}\u{1F1E0}-\u{1F1FF}\u{2600}-\u{26FF}\u{2700}-\u{27BF}\u{1F900}-\u{1F9FF}]/gu
  ```

#### 2. 验证逻辑优化
- **原始问题**: 错误的正则表达式范围导致中文字符被当作emoji过滤，过滤后变成空字符串
- **修复后**: 使用标准的emoji字符范围，只过滤真正的emoji字符，保留中文字符
- **适用场景**: 适用于所有使用 `validateField` 方法验证的字段

### 验证规则
- **长度校验**: 1-100 位字符（对于"名称"字段）
- **字符校验**: 允许中文、英文、日文、数字、空格，常见符号
- **特殊处理**: 
  - 自动过滤 emoji 字符
  - 当输入的内容全部为 emoji 时，提示"不能只包含特殊字符"
  - 保持其他验证规则不变

### 技术实现
- 修正了 emoji 过滤的正则表达式，使用标准的 emoji 字符范围
- 确保中文字符不被错误过滤
- 保持了原有的验证逻辑和错误提示

### 影响范围
- 所有使用 `validateField` 方法进行验证的字段，包括：
  - 名称
  - 入手途径
  - 原型
  - 涂装
  - 原画
  - 作品
  - 制造商
  - 比例
  - 材质
  - 尺寸

### 优化效果
- 修复了中文字符被错误过滤的问题
- 确保了"我回复挂号费的"等中文内容能正确通过验证
- 保持了对emoji的正确过滤
- 提高了验证的准确性和用户体验
