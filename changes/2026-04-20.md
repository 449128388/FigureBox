## 修复手办详情页入手价格显示

### 问题背景
手办详情页（FigureDetail）中显示的入手价格存在问题：
- 显示的是 `purchase_price`（旧的入手价格字段）
- 而不是 `average_purchase_price`（根据订单计算的平均入手价格）
- 导致详情页和编辑页面的价格显示不一致

1. **后端**：`GET /api/figures/{id}` 接口已正确返回 `average_purchase_price` 字段
2. **前端**：`FigureBasicInfo.vue` 组件中显示的是 `figure.purchase_price`
3. **结果**：详情页显示旧价格，编辑页显示平均价格，两者不一致

### 修复内容

**1. FigureBasicInfo.vue 组件更新**
- 将入手价格显示逻辑改为优先显示 `average_purchase_price`
- 如果没有平均入手价格（为0或未定义），则回退显示 `purchase_price`
- 标签相应改为"平均入手价格"或"入手价格"
- 平均价格保留2位小数显示

**代码修改**:
```vue

<div class="info-item" v-else-if="figure.purchase_price !== null && figure.purchase_price !== undefined && figure.purchase_price > 0">
  <span class="label">入手价格:</span>
  <span class="value">{{ figure.purchase_price }} {{ getCurrencySymbol(figure.purchase_currency) }}</span>
```

## 实现订单币种转换，统一计算入手价格为人民币

### 需求背景
当添加订单时，如果在定金或尾款的币种选择中选择了非人民币（如日元、美元、欧元），手办管理中计算入手价格时需要统一转换为人民币计算，确保价格比较的一致性。

### 汇率设置
- 1人民币 = 23日元
- 1美元 = 7人民币
- 1欧元 = 8人民币

### 后端修改

**1. Figure 模型新增币种转换方法**
- 新增 `_convert_to_cny` 方法，实现不同币种到人民币的转换
- 更新 `calculate_average_purchase_price` 方法，在计算平均价格时自动转换币种

**2. FigureService 列表查询优化**
- 在 `get_figures_list` 方法中添加币种转换逻辑
- 确保手办卡片显示的入手价格也是转换后的人民币价格

**代码修改**:
```python
# Figure 模型中的币种转换方法
def _convert_to_cny(self, amount, currency):
    """
    将金额转换为人民币
    
    汇率：
    - 1人民币 = 23日元
    - 1美元 = 7人民币
    - 1欧元 = 8人民币
    """
    if not amount:
        return 0

    exchange_rates = {
        'CNY': 1.0,    # 人民币
        'JPY': 1/23,   # 日元
        'USD': 7.0,    # 美元
        'EUR': 8.0     # 欧元
    }

    rate = exchange_rates.get(currency, 1.0)
    return amount * rate
```

- `backend/app/services/figure_service.py` - 更新列表查询中的价格计算逻辑

### 优化效果
- ✅ 支持多币种订单的入手价格计算
- ✅ 自动将不同币种转换为人民币
- ✅ 手办卡片和详情页显示统一的人民币价格
- ✅ 汇率固定，计算结果稳定

## 修复订单操作后手办平均入手价格未保存问题

### 问题背景
在手办管理中，平均入手价格是根据关联订单自动计算的，但之前的逻辑存在问题：
- 订单创建、更新、删除后，平均入手价格只在查询时实时计算
- 计算结果没有保存到数据库的 `average_purchase_price` 字段
- 导致手办列表、详情页显示的平均价格可能不一致或延迟更新

### 解决方案
采用企业级服务层架构，在订单相关操作后自动更新手办的平均入手价格。

### 后端修改

**1. FigureService 新增方法**
- 新增 `_convert_to_cny` 静态方法：实现币种到人民币的转换
- 新增 `update_figure_average_purchase_price` 静态方法：
  - 根据手办ID获取所有未软删除的订单
  - 将各订单的定金和尾款转换为人民币
  - 计算平均值并保存到数据库
  - 支持多币种混合计算

**2. Orders API 集成更新**
- 订单创建后：调用 `update_figure_average_purchase_price` 更新手办价格
- 订单更新后：调用 `update_figure_average_purchase_price` 更新手办价格
- 订单删除后：调用 `update_figure_average_purchase_price` 更新手办价格

**代码修改**:
```python
@staticmethod
def update_figure_average_purchase_price(db: Session, figure_id: int) -> float:
    """
    更新手办的平均入手价格
    
    根据关联的未软删除订单计算平均入手价格，并保存到数据库
    """
    from app.models.order import Order
    
    # 获取手办
    figure = db.query(Figure).filter(Figure.id == figure_id).first()
    if not figure:
        return 0
    
    # 获取所有未软删除的订单
    orders = db.query(Order).filter(
        Order.figure_id == figure_id,
        Order.is_active == 1
    ).all()
    
    # 计算平均入手价格
    if orders:
        total_amount = 0
        for order in orders:
            deposit_cny = FigureService._convert_to_cny(
                order.deposit or 0, 
                order.deposit_currency or 'CNY'
            )
            balance_cny = FigureService._convert_to_cny(
                order.balance or 0, 
                order.balance_currency or 'CNY'
            )
            total_amount += deposit_cny + balance_cny
        average_price = total_amount / len(orders)
    else:
        average_price = 0
    
    # 更新手办的平均入手价格
    figure.average_purchase_price = average_price
    db.commit()
    db.refresh(figure)
    
    return average_price
```

```python
# 重构前（get_figures_list 方法内部）
def _convert_to_cny(amount, currency):
    """
    将金额转换为人民币

    汇率：
    - 1人民币 = 23日元
    - 1美元 = 7人民币
    - 1欧元 = 8人民币
    """
    if not amount:
        return 0

    exchange_rates = {
        'CNY': 1.0,    # 人民币
        'JPY': 1/23,   # 日元
        'USD': 7.0,    # 美元

    rate = exchange_rates.get(currency, 1.0)
    return amount * rate
```
