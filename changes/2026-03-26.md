## 2. 修复下载接口字段名错误

### 问题原因
- 下载接口中使用了错误的字段名：`final_payment` 和 `release_date`
- Order 模型的实际字段名为：`balance` 和 `due_date`
- 导致 500 错误：`'Order' object has no attribute 'final_payment'`

### 解决方案
- 修正字段名：`final_payment` → `balance`
- 修正字段名：`release_date` → `due_date`
- 移除不存在的字段：`created_at` 和 `updated_at`

### 功能描述
将手办管理页面的搜索功能从**前端本地过滤**改为**后端接口查询**，实现前后端功能分离，提升大数据量下的搜索性能。

```python
@router.get("/", response_model=list[FigureSchema])
def get_figures(
    skip: int = 0,
    limit: int = 100,
    name: str = None,
    purchase_type: str = None,
    purchase_date_start: str = None,
    purchase_date_end: str = None,
    db: Session = Depends(get_db)
):
    """
    获取手办列表，支持搜索过滤
    """
    query = db.query(Figure)
    
    # 按名称搜索（模糊匹配）
    if name:
        query = query.filter(Figure.name.ilike(f"%{name}%"))
    
    # 按入手形式过滤
    if purchase_type:
        query = query.filter(Figure.purchase_type == purchase_type)
    
    # 按入手日期范围过滤
    if purchase_date_start:
        try:
            start_date = datetime.strptime(purchase_date_start, "%Y-%m-%d").date()
            query = query.filter(Figure.purchase_date >= start_date)
        except ValueError:
            pass
    
    if purchase_date_end:
        try:
            end_date = datetime.strptime(purchase_date_end, "%Y-%m-%d").date()
            query = query.filter(Figure.purchase_date <= end_date)
        except ValueError:
            pass
    
    # 按 id 降序排序（最新的在前面）
    query = query.order_by(Figure.id.desc())
    
    figures = query.offset(skip).limit(limit).all()
    return figures
```

## 4. 修复搜索时重复请求问题

### 问题原因
在 `frontend/src/store/index.js` 的 `fetchFigures` 方法中，当有搜索条件时，代码会发送两次相同的 HTTP 请求：

```javascript
const response = await axios.get(url)  // 第一次请求
this.figures = response

// 如果有搜索条件，同时获取总数（用于分页）
if (Object.keys(params).length > 0 && ...) {
  const countResponse = await axios.get(url)  // 第二次请求（重复）
  this.totalCount = countResponse.length
}
```

这是一个逻辑错误，第二次请求是多余的，因为：
1. 两次请求的 URL 完全相同
2. 返回的数据也相同
3. `totalCount` 可以直接使用第一次请求的结果

### 解决方案
移除重复的第二次请求，直接使用第一次请求的结果设置 `totalCount`：

```javascript
const response = await axios.get(url)
this.figures = response

// 设置总数（用于分页）
this.totalCount = response.length
```

### 变更内容
**文件**: `frontend/src/store/index.js`

**修复前**:
```javascript
async fetchFigures(params = {}) {
  // ... 构建 URL ...
  
  const response = await axios.get(url)
  this.figures = response
  
  // 如果有搜索条件，同时获取总数（用于分页）
  if (Object.keys(params).length > 0 && (params.name || params.purchase_type || params.purchase_date_start)) {
    const countResponse = await axios.get(url)  // 重复请求
    this.totalCount = countResponse.length
  } else {
    this.totalCount = response.length
  }
}
```

**修复后**:
```javascript
async fetchFigures(params = {}) {
  // ... 构建 URL ...
  
  const response = await axios.get(url)
  this.figures = response
  
  // 设置总数（用于分页）
  this.totalCount = response.length
}
```

- ✅ 修复了搜索时发送 2 次请求的问题
- ✅ 现在每次搜索只发送 1 次请求

- ✅ 提升搜索响应速度

### 性能优化

- **降低服务器压力**：减少 50% 的查询负载
- **提升响应速度**：减少网络延迟
