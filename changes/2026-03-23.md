## 2. 尾款管理编辑订单新增物流订单字段

### 变更原因
- 用户需要记录订单的物流信息以便跟踪包裹
- 物流订单号只在订单创建后才有，适合在编辑时添加
- 新增订单时还没有物流信息，不需要此字段

### 变更内容

#### 后端更新
**文件**: `backend/app/models/order.py`
- 添加 `tracking_number` 字段（VARCHAR(100)，可为空）
- 用于存储物流订单号

**文件**: `backend/app/schemas/order.py`
- `OrderUpdate` schema 添加 `tracking_number` 字段
- `Order` schema 添加 `tracking_number` 字段

#### 数据库更新
- 执行 SQL：`ALTER TABLE orders ADD COLUMN tracking_number VARCHAR(100) AFTER shop_contact;`

#### 前端更新
**文件**: `frontend/src/views/Orders.vue`
- 编辑订单表单添加"物流订单"输入框
- 使用 `v-if="isEditing"` 只在编辑模式下显示
- 重置表单时包含 `tracking_number` 字段

### 功能效果
- 编辑订单时可以输入物流订单号
- 新增订单时不显示物流订单字段
- 物流订单号数据会保存到数据库

### 用户体验优化
- **信息完整**: 可以记录和查看物流信息
- **操作合理**: 只在需要时（编辑）显示物流字段
- **流程顺畅**: 新增订单不被无关字段干扰

## 3. 物流订单信息展示优化

### 变更原因
- 物流订单号已可在编辑订单时录入，但无法在列表和详情页查看
- 需要在尾款管理列表和手办详情页展示物流订单信息
- 无物流订单号时不应显示，保持页面简洁

### 变更内容

#### 前端更新
**文件**: `frontend/src/views/Orders.vue`
- 在订单列表项中添加物流订单展示
- 使用 `v-if="order.tracking_number"` 只在有数据时显示

**文件**: `frontend/src/views/FigureDetail.vue`
- 在尾款信息模块中添加物流订单展示
- 使用 `v-if="relatedOrder.tracking_number"` 只在有数据时显示

### 功能效果
- 尾款管理列表中，有物流订单号的订单会显示该信息
- 手办详情页的尾款信息模块中，会显示关联订单的物流订单号
- 无物流订单号时，该字段自动隐藏，不占用空间

### 用户体验优化
- **信息可见**: 可以在列表和详情页查看物流订单号
- **智能显示**: 有数据时显示，无数据时隐藏，保持页面简洁
- **便于跟踪**: 方便用户跟踪包裹物流信息
